### 一、 技术架构与大模型应用 (Technical Path)

**核心挑战：** 评委担心本科生团队在“开发大模型”上浪费时间，或技术实现无法支撑实验控制。

| 可能的问题 | 应对思路（核心：后端中台控变量） |
| --- | --- |
| **1. 你们是自己开发大模型吗？可行性如何？** | **明确界定：** 我们不从零训练大模型。我们采用“领域适配型集成开发”：调用成熟 API（如 DeepSeek-V3）作为生成引擎，自研 **Python/Django 后端**作为实验管理中台，负责逻辑控制与数据采集。 |
| **2. API 调用在架构中如何实现风格控制？** | **动态注入机制：** 在后端 Django 层根据用户分组，动态拼接不同的 **System Prompt**（如：要求 AI 多用语气词或严禁情感反馈）。同时控制前端 **“正在输入”** 的动画时长，从言语和非言语两个维度操纵变量。 |
| **3. 如果 API 不稳定或技术实现受阻怎么办？** | **备选方案：** 采用 **Wizard of Oz (奥兹国大王法)**，利用现成的 Agent 平台（如 Coze/Dify）预设 Prompt 模板。若自动化失败，则由研究者后台选择回复逻辑，确保实验流程不断档。 |
| **4. 数据隐私与安全性如何保证？** | **后端脱敏：** API 调用放在服务端而非客户端，API Key 绝不暴露。所有对话在流向云端前进行敏感信息脱敏，并预留校内实验室服务器（如 Ollama 部署）的私有化接口。 |

### 二、 研究方法与实验设计 (Methodology)

**核心挑战：** 评委关注实验的严谨性、样本的代表性以及“计算社会科学”名词是否空泛。

| 可能的问题 | 应对思路（核心：逻辑递进与因果验证） |
| --- | --- |
| **1. 你们的三阶段实验是独立的还是递进的？** | **链式递进：** 阶段一验证“风格与表露（入口）”，阶段二验证“干预与重塑（核心）”，阶段三验证“关系与依恋（结果）”。前一阶段的产出（如表露的关键词）是后一阶段干预的素材。 |
| **2. AI 的回应风格如何做到标准化控制？** | **锁定参数：** 在 API 调用时锁定 `Temperature`（温度系数）为低值（0.3-0.5），并使用 **Few-shot Prompting**（在指令中加入标准示例）来减少 AI 输出的随机性，确保同组风格高度一致。 |
| **3. 样本量够吗？如何招募？** | **量化依据：** 使用 G*Power 测算，设定中等效应值，三阶段预计需 200-300 名有效样本。**渠道：** 通过校内心理课程积分激励、学生处社群精准招募，确保样本具备代表性。 |
| **4. “计算社会科学范式”体现在哪？** | **双重定义：** 1. **计算传播：** 对对话文本进行语义网络分析；2. **计算心理：** 将用户情感极性转化为 AI 强化学习的 **Reward 奖励信号**，实现干预路径的自动优化。 |

### 三、 理论深度与心理学伦理 (Theory & Ethics)

**核心挑战：** 评委考察跨学科结合的深度，以及对心理健康这一敏感领域的风险防控。

| 可能的问题 | 应对思路（核心：双向互构与安全红线） |
| --- | --- |
| **1. 什么是“AI 与心理学的双向互构”？** | **闭环逻辑：** 心理学理论（如认知重评）指导 AI 的干预策略；AI 的交互数据（强化学习反馈）反过来揭示大学生心理变化的动态规律，修正传统静态干预模型的不足。 |
| **2. 如何评估 AI 引导用户“自我探索”的效果？** | **指标量化：** 除了 GAD-7（焦虑）、PHQ-9（抑郁）等结果量表，我们自研了 **“过程评估量表”**，从动态适配、认知觉察、数字依恋三个维度测量干预过程的质量。 |
| **3. 如果用户在对话中出现自残等倾向怎么办？** | **安全红线：** 系统内置敏感词过滤和情绪危机识别逻辑。一旦检测到高危信号，系统立即中断 AI 自由对话，强制推送人工干预接口和学校心理中心热线，并对研究团队预警。 |
| **4. AI 的“温暖”和“职业边界”如何平衡？** | **适度拟人：** 我们的研究正是要探索这个“度”。通过对比实验，寻找既能促进表露又不至于引发病理性依赖的最佳拟人化水平（CASA 范式的边界应用）。 |
